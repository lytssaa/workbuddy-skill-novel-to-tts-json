# Fish Audio S2.1 Pro 有声书脚本转换规范（精简版）

> 每章转换前必须读取此文件全部内容。以下是完整的转换规则，严格遵守每一条。

请仔细阅读提供的小说文本，将其转换为适合 Fish Audio S2.1 Pro TTS 引擎的有声书脚本 JSON 格式。Fish Audio S2.1 Pro 的特点：通过参考音频克隆音色，支持局部语气标注，文本质量直接决定音质。

**注意**：TXT 文件头尾可能包含播客平台用语（如 `欢迎您收听由喜马拉雅出品…` `听众朋友，本集已播讲完毕…`），这些是文件完整组成部分，需要一并纳入 JSON 转换。

## 一、输出结构规范

输出根对象包含两个字段：

- `character_map`: (对象) 角色画像
- `script`: (数组) 演绎脚本

## 二、角色分析 (character_map)

提取关键角色（不含旁白），每个角色包含且仅包含以下五个字段：

| 字段 | 类型 | 说明 |
|------|------|------|
| `gender` | string | "男" / "女" |
| `age` | string | 如 "中年"、"20岁左右"、"六旬开外" |
| `role_tag` | string | "重要角色" 或 "路人角色" |
| `personality` | string | 重要角色 30-60 字画像；路人角色 20-30 字简述 |
| `timbre` | string | 重要角色描述辨识度特征；路人角色统一写"普通路人中性声线，无特殊辨识度" |

**⚠️ 这五个字段一个都不能少，也不能多。** 不要用 `tone`、`description`、`voice` 等替代字段名。

## 三、script 字段（必须严格遵守！）

每个对象包含且仅包含以下**三个**字段：

1. `speaker`: "旁白" 或角色名
2. `content`: 台词或旁白内容（见第五章详细规则）
3. `delay`: 毫秒数（必须使用第四章的默认值，禁止随意填）

⚠️ **不要输出 emo_vector 和 ref_emotion，系统会自动补充。**

## 四、delay 强制默认值（禁止随意填写！）

| 场景 | delay 值 | 区间 |
|------|---------|------|
| 旁白段落之间 | 800ms | 600-1000 |
| 角色对话前置 | 500ms | 400-800 |
| 情节转折/紧张 | 1500ms | 1200-2000 |
| 场景切换 | 2000ms | 1500-2500 |
| 对话轮换（不同角色之间） | 500ms | — |

## 五、content 文本规则

### 5.1 长度限制（强制！）

- 单条 content 纯汉字（去掉 `[标签]` 后）必须 ≤120 字
- 超过 120 字必须拆分为多条 script 对象
- 拆分点：在标点处断开，保持语义完整

### 5.2 局部语气标注（强制要求！）

⚠️ **角色台词（speaker ≠ "旁白"）必须加语气标签，旁白不加。** 标签用 Fish Audio 官方英文标签，不要用中文标签。

**标签对照表（Fish Audio 官方英文标签）：**

| 标签 | 适用场景 | 示例台词 |
|------|---------|---------|
| `[whisper]` | 耳语、密谋、悄悄话 | "这件事我不会就此作罢。" |
| `[low voice]` | 低声、平静、自言自语、平缓对话 | "在下姓林，草字廷扬。" |
| `[angry]` | 怒骂、斥责、愤恨 | "你以为我会怕你吗！" |
| `[sad]` | 哭泣、哀叹、丧亲 | "想不到昨日一别，竟成永诀！" |
| `[chuckle]` | 冷笑、嘲讽、讥讽 | "就凭你，也配来叫阵？" |
| `[surprised]` | 惊讶、意外 | "他……他真的死了吗？" |
| `[shocked]` | 震惊、不敢相信 | "怎么，林大哥竟会死在贼人手里！" |
| `[emphasis]` | 强调、严肃、一字一顿 | "此仇非报不可。" |
| `[excited]` | 激动、兴奋、叫阵 | "就请赐招！" |
| `[shouting]` | 大喊、叫喊、呼喝 | "住手！" |
| `[sigh]` | 叹气、无奈、遗憾 | "这就是我们干镖行的收缘结果。" |
| `[delight]` | 高兴、喜悦、欣慰 | "好极了！" |
| `[panting]` | 喘息、紧张、急促 | "快……快走！" |
| `[pause]` | 停顿、沉吟、犹豫 | "这个嘛……容我想想。" |
| `[loud]` | 大声、洪亮 | "好！说得好！" |
| `[volume down]` | 渐弱、压低声音 | "今晚贼人必不肯轻易放过。" |

**自检：** 输出前检查所有非旁白台词，每条都必须带 `[xxx]` 标签。

### 5.3 文本优化

- **断句自然**：语义完整短语间用逗号分隔
- **节奏感**：短句和长句交替
- **口语化**：书面长句拆成口语短句
- **标点**：只保留 ，、。！？和……

### 5.4 ⚠️ 禁止摘要压缩（绝对红线）

**这是逐句转换任务，不是摘要任务。** TXT 中每一句话都必须在 JSON 中有对应的 content 条目。

- ❌ 不得把多段叙述合并成一句话带过
- ❌ 不得省略人物特长描写、场景细节、善后安排等"非对话"段落
- ❌ 不得因为某段"没有台词"就跳过不转
- ✅ 叙述性旁白也必须逐句转换为 `"speaker": "旁白"` 的 content 条目
- ✅ 如果一段叙述超过 120 字，在标点处断成多条 content，但内容不能丢

**自检方法**：输出前拼接所有 content 的汉字数，与原文汉字数对比。低于 92% = 有遗漏，必须补。

### 5.5 speaker 归属（绝对红线）

- 引号外的所有字 → 归"旁白"
- 引号内的所有字 → 归角色（去掉引号）
- **夹心句式必须拆分**：台词A → 旁白动作 → 台词B

### 5.6 角色名称一致性

- 同一角色在全文中统一使用一个名称，不要混用别名
- 首次出场就应记录到 character_map，后面复用即可

## 六、输出纯净性

输出纯净 JSON，严禁 markdown 标记 ` ```json ` 包裹，严禁在 JSON 前后添加任何解释性文字。

## 七、模板示例

```json
{
  "character_map": {
    "李昂": {"gender": "男", "age": "38岁左右", "role_tag": "重要角色", "personality": "表面颓废实则深情。", "timbre": "磁性中年大叔音，声线低沉略带烟嗓。"},
    "店小二": {"gender": "男", "age": "20岁左右", "role_tag": "路人角色", "personality": "服务行业人员，热情圆滑。", "timbre": "普通路人中性声线，无特殊辨识度"}
  },
  "script": [
    {"speaker": "旁白", "content": "夜色如墨，寒风呼啸。", "delay": 800},
    {"speaker": "店小二", "content": "[low voice]客官，要住店吗？", "delay": 500},
    {"speaker": "李昂", "content": "[whisper]不用了。", "delay": 500},
    {"speaker": "旁白", "content": "他头也不回，继续往前走。", "delay": 800}
  ]
}
```

## 八、转换后处理

转换完成并 check 通过后，在终端执行以下命令补字段：

```bash
# 单文件
python scripts/fix_all.py output/联镖记_020_xxx.json

# 批量（推荐，修复所有新生成的JSON）
python scripts/fix_all.py output/
```

`fix_all.py` 会：
1. 修复 content 中的未转义中文引号
2. 自动补上 emo_vector（全0数组）
3. 自动补上 ref_emotion（"中性"）

## 九、常见错误自查表（输出前逐项核对）

| # | 检查项 | 正确 | 错误示例 |
|---|--------|------|---------|
| 1 | character_map 字段名 | `gender`, `age`, `role_tag`, `personality`, `timbre` | 使用 `tone`, `voice`, `description` |
| 2 | role_tag 值 | `"重要角色"` 或 `"路人角色"` | `"主要角色"`, `"重要"` |
| 3 | content 长度 | 去标签后 ≤120 汉字 | 整段 200+ 字不拆分 |
| 4 | speaker 归属 | 引号外→旁白，引号内→角色 | 把动作叙述写成角色台词 |
| 5 | 夹心句式 | 拆成 A→旁白→B | 全部塞进一条 |
| 6 | JSON 纯净性 | 纯 JSON，无包裹 | ` ```json ... ``` ` |
| 7 | 播客头尾 | 已纳入 JSON | 擅自剥离播客头尾 |
| 8 | 角色名一致 | 全文同一角色同名词 | 林廷扬/总镖头/狮子林 混用 |
| 9 | 台词语气标签 | 非旁白必须带 `[xxx]` 标签 | 全部台词无标签，平淡如水 |
| 10 | 三字段完整 | speaker, content, delay 全有 | 漏掉 delay |

```

{章节文本}
```
